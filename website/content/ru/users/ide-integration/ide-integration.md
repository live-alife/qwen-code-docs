# Интеграция с IDE

Qwen Code может интегрироваться с вашей IDE, обеспечивая более плавный и контекстно-зависимый опыт работы. Эта интеграция позволяет CLI лучше понимать ваше рабочее пространство и включает мощные функции, такие как нативное сравнение изменений (diff) прямо в редакторе.

В настоящее время единственной поддерживаемой IDE является [Visual Studio Code](https://code.visualstudio.com/) и другие редакторы, поддерживающие расширения VS Code. Чтобы добавить поддержку других редакторов, ознакомьтесь со [спецификацией расширения-компаньона для IDE](../ide-integration/ide-companion-spec).

## Возможности

- **Контекст рабочего пространства:** CLI автоматически получает информацию о вашем рабочем пространстве, чтобы предоставлять более релевантные и точные ответы. Этот контекст включает:
  - **10 последних открытых файлов** в вашем рабочем пространстве.
  - Позицию активного курсора.
  - Любой выделенный вами текст (до 16 КБ; более длинные выделения будут обрезаны).

- **Нативное сравнение (Diff):** Когда Qwen предлагает изменения в коде, вы можете просматривать их прямо во встроенном средстве сравнения вашей IDE. Это позволяет удобно просматривать, редактировать, принимать или отклонять предложенные изменения.

- **Команды VS Code:** Вы можете запускать функции Qwen Code прямо из палитры команд VS Code (`Cmd+Shift+P` или `Ctrl+Shift+P`):
  - `Qwen Code: Run`: Запускает новую сессию Qwen Code во встроенном терминале.
  - `Qwen Code: Accept Diff`: Принимает изменения в активном редакторе сравнения.
  - `Qwen Code: Close Diff Editor`: Отклоняет изменения и закрывает активный редактор сравнения.
  - `Qwen Code: View Third-Party Notices`: Отображает уведомления о сторонних компонентах для расширения.

## Установка и настройка

Существует три способа настройки интеграции с IDE:

### 1. Автоматическое предложение (Рекомендуется)

При запуске Qwen Code в поддерживаемом редакторе он автоматически обнаружит вашу среду и предложит подключиться. Ответ «Yes» автоматически запустит необходимую настройку, включая установку расширения-компаньона и активацию соединения.

### 2. Ручная установка через CLI

Если вы ранее отклонили предложение или хотите установить расширение вручную, выполните следующую команду внутри Qwen Code:

```
/ide install
```

Команда найдет подходящее расширение для вашей IDE и установит его.

### 3. Ручная установка из маркетплейса

Вы также можете установить расширение напрямую из маркетплейса.

- **Для Visual Studio Code:** Установите из [маркетплейса VS Code](https://marketplace.visualstudio.com/items?itemName=qwenlm.qwen-code-vscode-ide-companion).
- **Для форков VS Code:** Для поддержки форков VS Code расширение также опубликовано в [реестре Open VSX](https://open-vsx.org/extension/qwenlm/qwen-code-vscode-ide-companion). Следуйте инструкциям вашего редактора по установке расширений из этого реестра.

> NOTE:
> Расширение "Qwen Code Companion" может отображаться в нижней части результатов поиска. Если вы не видите его сразу, попробуйте прокрутить вниз или отсортировать по "Newly Published".
>
> После ручной установки расширения необходимо выполнить `/ide enable` в CLI для активации интеграции.

## Использование

### Включение и отключение

Вы можете управлять интеграцией с IDE прямо из CLI:

- Чтобы включить соединение с IDE, выполните:
  ```
  /ide enable
  ```
- Чтобы отключить соединение, выполните:
  ```
  /ide disable
  ```

После включения Qwen Code будет автоматически пытаться подключиться к расширению-компаньону IDE.

### Проверка статуса

Чтобы проверить статус соединения и увидеть контекст, полученный CLI от IDE, выполните:

```
/ide status
```

При успешном подключении команда покажет IDE, к которой установлено соединение, и список недавно открытых файлов, о которых она знает.

(Примечание: список файлов ограничен 10 последними открытыми файлами в вашем рабочем пространстве и включает только локальные файлы на диске.)

### Работа с изменениями (Diff)

Когда вы просите модель Qwen изменить файл, она может открыть представление сравнения (diff) прямо в вашем редакторе.

**Чтобы принять изменения (diff)**, выполните одно из следующих действий:

- Нажмите на **значок галочки** в заголовке редактора сравнения.
- Сохраните файл (например, с помощью `Cmd+S` или `Ctrl+S`).
- Откройте палитру команд и выполните **Qwen Code: Accept Diff**.
- Введите `yes` в CLI при появлении запроса.

**Чтобы отклонить изменения (diff)**, вы можете:

- Нажмите на **значок 'x'** в заголовке редактора сравнения.
- Закрыть вкладку редактора сравнения.
- Открыть палитру команд и выполнить **Qwen Code: Close Diff Editor**.
- Ввести `no` в CLI при появлении запроса.

Вы также можете **отредактировать предложенные изменения** прямо в представлении сравнения перед их принятием.

Если вы выберете «Yes, allow always» в CLI, изменения больше не будут отображаться в IDE, так как будут приниматься автоматически.

## Использование в песочнице (Sandboxing)

Если вы используете Qwen Code в песочнице, обратите внимание на следующее:

- **На macOS:** Интеграция с IDE требует сетевого доступа для связи с расширением-компаньоном. Необходимо использовать профиль Seatbelt, разрешающий сетевой доступ.
- **В контейнере Docker:** Если вы запускаете Qwen Code внутри контейнера Docker (или Podman), интеграция с IDE всё равно сможет подключиться к расширению VS Code, работающему на хост-машине. CLI настроен на автоматический поиск сервера IDE по адресу `host.docker.internal`. Обычно специальная настройка не требуется, но вам может потребоваться убедиться, что сетевая конфигурация Docker разрешает соединения из контейнера на хост.

## Устранение неполадок

Если вы столкнулись с проблемами при интеграции с IDE, ниже приведены распространенные сообщения об ошибках и способы их решения.

### Ошибки подключения

- **Сообщение:** `🔴 Disconnected: Failed to connect to IDE companion extension for [IDE Name]. Please ensure the extension is running and try restarting your terminal. To install the extension, run /ide install.`
  - **Причина:** Qwen Code не смог найти необходимые переменные окружения (`QWEN_CODE_IDE_WORKSPACE_PATH` или `QWEN_CODE_IDE_SERVER_PORT`) для подключения к IDE. Обычно это означает, что расширение-компаньон IDE не запущено или не инициализировалось корректно.
  - **Решение:**
    1. Убедитесь, что вы установили расширение **Qwen Code Companion** в вашей IDE и оно включено.
    2. Откройте новое окно терминала в вашей IDE, чтобы оно подхватило корректное окружение.

- **Сообщение:** `🔴 Disconnected: IDE connection error. The connection was lost unexpectedly. Please try reconnecting by running /ide enable`
  - **Причина:** Соединение с расширением-компаньоном IDE было потеряно.
  - **Решение:** Выполните `/ide enable`, чтобы попытаться переподключиться. Если проблема сохраняется, откройте новое окно терминала или перезапустите IDE.

### Ошибки конфигурации

- **Сообщение:** `🔴 Disconnected: Directory mismatch. Qwen Code is running in a different location than the open workspace in [IDE Name]. Please run the CLI from the same directory as your project's root folder.`
  - **Причина:** Текущий рабочий каталог CLI находится за пределами папки или рабочего пространства, открытого в вашей IDE.
  - **Решение:** Перейдите в тот же каталог, который открыт в вашей IDE, с помощью `cd` и перезапустите CLI.

- **Сообщение:** `🔴 Disconnected: To use this feature, please open a workspace folder in [IDE Name] and try again.`
  - **Причина:** В вашей IDE не открыто рабочее пространство.
  - **Решение:** Откройте рабочее пространство в вашей IDE и перезапустите CLI.

### Общие ошибки

- **Сообщение:** `IDE integration is not supported in your current environment. To use this feature, run Qwen Code in one of these supported IDEs: [List of IDEs]`
  - **Причина:** Вы запускаете Qwen Code в терминале или среде, которая не является поддерживаемой IDE.
  - **Решение:** Запускайте Qwen Code из встроенного терминала поддерживаемой IDE, например, VS Code.

- **Сообщение:** `No installer is available for IDE. Please install the Qwen Code Companion extension manually from the marketplace.`
  - **Причина:** Вы выполнили `/ide install`, но CLI не имеет автоматического установщика для вашей конкретной IDE.
  - **Решение:** Откройте маркетплейс расширений вашей IDE, найдите "Qwen Code Companion" и установите его вручную.
