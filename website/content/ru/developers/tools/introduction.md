# Инструменты Qwen Code

Qwen Code включает набор встроенных инструментов, с помощью которых модель взаимодействует с вашей локальной средой, получает нужную информацию и выполняет действия. Эти инструменты расширяют возможности CLI: он не просто генерирует текст, а помогает выполнять реальные задачи в проекте.

## Обзор инструментов Qwen Code

В Qwen Code инструмент — это отдельная функция или модуль, выполнение которого модель может запросить во время работы. Например, если вы попросите модель «суммировать содержимое `my_document.txt`», она, скорее всего, поймет, что файл нужно прочитать, и запросит вызов инструмента `read_file`.

Основной компонент (`packages/core`) управляет этими инструментами: передает модели их описания и schema, выполняет инструмент после запроса и возвращает результат обратно модели, чтобы она могла сформировать понятный ответ для пользователя.

Инструменты дают Qwen Code следующие возможности:

- **Доступ к локальной информации:** модель может работать с вашей файловой системой, читать содержимое файлов, просматривать каталоги и выполнять похожие операции.
- **Выполнение команд:** с помощью инструментов вроде `run_shell_command` модель может запускать shell-команды с учетом защитных механизмов и, когда это требуется, с подтверждением пользователя.
- **Работа с Web:** инструменты могут получать содержимое по URL.
- **Выполнение действий:** инструменты могут изменять файлы, создавать новые файлы и выполнять другие операции в вашей системе. Для таких действий обычно применяются дополнительные меры безопасности.
- **Опора на фактический контекст:** когда модель получает актуальные данные из локальной среды или внешних источников, ответы становятся точнее, релевантнее и лучше привязаны к вашему реальному проекту.

## Как использовать инструменты Qwen Code

Чтобы задействовать инструменты Qwen Code, достаточно отправить prompt в CLI. Процесс выглядит так:

1.  Вы вводите prompt в CLI.
2.  CLI передает prompt основному компоненту.
3.  Основной компонент отправляет в настроенный model API ваш prompt, историю диалога, а также список доступных инструментов с их описаниями и schema.
4.  Модель анализирует запрос. Если для ответа нужен инструмент, она возвращает запрос на вызов конкретного инструмента с нужными параметрами.
5.  Основной компонент получает этот запрос, проверяет его и выполняет инструмент. Для чувствительных операций обычно требуется подтверждение пользователя.
6.  Результат выполнения инструмента отправляется обратно модели.
7.  Модель использует этот результат, чтобы сформировать финальный ответ. Затем ответ возвращается через основной компонент в CLI и отображается пользователю.

В CLI обычно отображаются сообщения о том, когда вызывается инструмент и успешно ли завершилось выполнение.

## Безопасность и подтверждения

Многие инструменты, особенно те, которые могут изменять файловую систему или выполнять команды (`write_file`, `edit`, `run_shell_command`), спроектированы с учетом безопасности. Qwen Code обычно:

- **Запрашивает подтверждение:** перед потенциально чувствительным действием показывает, что именно будет выполнено, и просит подтвердить операцию.
- **Использует sandbox-изоляцию:** все инструменты работают с учетом ограничений sandbox-механизма (см. [Sandbox в Qwen Code](../sandbox.md)). Это означает, что при работе внутри sandbox любые нужные инструменты, включая MCP-серверы, должны быть доступны _внутри_ sandbox-среды. Например, чтобы запустить MCP-сервер через `npx`, исполняемый файл `npx` должен быть установлен в Docker-образе sandbox или доступен в окружении `sandbox-exec`.

Перед тем как разрешить инструменту продолжить работу, внимательно проверяйте запрос на подтверждение.

## Подробнее об инструментах Qwen Code

Встроенные инструменты Qwen Code можно условно разделить на несколько категорий:

- **[Инструменты файловой системы](./file-system.md):** чтение, запись, просмотр, поиск и другие операции с файлами и каталогами.
- **[Shell-инструмент](./shell.md) (`run_shell_command`):** выполнение shell-команд.
- **[Инструмент Web Fetch](./web-fetch.md) (`web_fetch`):** получение содержимого по URL.
- **[Инструмент чтения нескольких файлов](./multi-file.md) (`read_many_files`):** специализированный инструмент для чтения содержимого нескольких файлов или каталогов. Часто используется командой `@`.
- **[Инструмент памяти](./memory.md) (`save_memory`):** сохранение и повторное использование информации между сессиями.
- **[Инструмент Todo Write](./todo-write.md) (`todo_write`):** создание и ведение структурированных списков задач во время coding-сессии.
- **[Инструмент Task](./task.md) (`task`):** делегирование сложных задач специализированным subagents.
- **[Инструмент Exit Plan Mode](./exit-plan-mode.md) (`exit_plan_mode`):** выход из plan mode и переход к реализации.

Кроме того, эти инструменты интегрируются со следующими возможностями:

- **[MCP-серверы](./mcp-server.md)**: MCP-серверы служат связующим слоем между моделью и вашей локальной средой или внешними сервисами, например API.
  - **[Краткое руководство по MCP](../mcp-quick-start.md)**: практические примеры, чтобы начать работу с MCP за 5 минут
  - **[Примеры конфигураций MCP](../mcp-example-configs.md)**: готовые конфигурации для типовых сценариев
  - **[Web Search через MCP](./web-search.md)**: подключение поисковых сервисов, таких как Bailian, Tavily и GLM, через MCP
  - **[Тестирование и валидация MCP](../mcp-testing-validation.md)**: проверка и валидация настроек MCP-серверов
- **[Sandbox](../sandbox.md)**: sandbox изолирует модель и ее изменения от вашей среды, снижая потенциальные риски.
