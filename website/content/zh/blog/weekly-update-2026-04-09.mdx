本周发布 **v0.14.2** 和 3 个 bugfix 版本。

这轮更新让 Qwen Code 的协作方式更灵活、接入场景也更多了：微信/Telegram/钉钉都能直接用了；`/plan` 让 AI 先出计划再动手；Ctrl+O 一键切换详细/简洁输出；Subagent 支持跨提供商选模型；会话内还能设定时任务让 AI 自动检查进度。

## ✨ 新增特性

### 多平台接入：把 Qwen Code 接入到微信/Telegram/钉钉

不再局限于终端和 VS Code，可以把 Qwen Code 接到微信/Telegram/钉钉这些通讯平台。在每个平台作为独立插件运行，需要用户配置和认证。

**你能用它做什么：**

- 在微信里直接问代码问题，不用打开终端
- 把 Telegram 当远程开发助手，手机上随时发指令
- 在钉钉里集成 AI 编码助手

详见 PR [#2628](https://github.com/QwenLM/qwen-code/pull/2628)

<img src="https://img.alicdn.com/imgextra/i3/O1CN01F5IFsE1DqnHMReKfd_!!6000000000268-0-tps-4032-2565.jpg" alt="多平台接入演示" class="max-w-4xl h-auto rounded-lg" />

### /plan 规划模式：先规划再执行

新增 `/plan` 命令。执行前先让 AI 理清思路、制定方案、列出步骤。规划完再进入执行模式，方向更清晰。

**你能用它做什么：**

- 大需求先用 `/plan` 让 AI 自己拆解任务，想清楚每一步再执行，不盲目上手
- 改 bug 前先让 AI 分析可能的原因和排查路径，不盲目试错
- 重构前先出完整方案，等你点头再动手改代码，不怕 AI 乱改

详见 PR [#2921](https://github.com/QwenLM/qwen-code/pull/2921)

### Subagent 支持跨提供商选模型

之前 Subagent 只能跟主会话用同一个模型和同一个 API 提供商。现在可以为每个 Subagent 单独指定模型和提供商，比如主会话用 Qwen，Subagent 用 GPT-4o，或者反过来。

**你能用它做什么：**

- 主会话用 Qwen 处理复杂推理，Subagent 用便宜的模型做文件搜索、格式转换等简单任务，省 token
- 某个 API 限流或出问题时，Subagent 切到备用提供商继续干活，不中断
- 对比不同模型在同一任务上的效果，选最优的那个

详见 PR [#2698](https://github.com/QwenLM/qwen-code/pull/2698)

<video src="https://cloud.video.taobao.com/vod/28mD0EPBjsGSq3CI6fxuVP86QknwjjqTYW7uwk00Iog.mp4" controls class="max-w-4xl h-auto rounded-lg" />

### Ctrl+O 切换详细/简洁模式

按 Ctrl+O 可以在详细模式和简洁模式之间快速切换，控制 AI 输出的详细程度。

**你能用它做什么：**

- 调试时切到详细模式，看 AI 的完整思考过程和工具调用
- 赶时间时切到简洁模式，只看结果不看中间过程
- 同一个会话里按需切换，不用重新配置

详见 PR [#2770](https://github.com/QwenLM/qwen-code/pull/2770)

<img src="https://img.alicdn.com/imgextra/i1/O1CN01Zod5j61YlOXUboPiH_!!6000000003099-1-tps-1251-884.gif" alt="Ctrl+O 切换详细/简洁模式" class="max-w-4xl h-auto rounded-lg" />

### 会话内定时任务：让 AI 定时自动检查

新增 `/loop` 命令，可以在当前会话里设一个定时任务。比如输入 `/loop 5m check the build`，AI 就会每隔 5 分钟自动检查一下构建状态，不用你反复问"部署完了没"。

**你能用它做什么：**

- 启动耗时任务后设个定时检查，部署/构建/数据迁移自动追踪进度
- 每隔一段时间检查一下 PR 有没有新反馈
- 任务只活在当前会话里，关掉就没了，不占资源

详见 PR [#2731](https://github.com/QwenLM/qwen-code/pull/2731)

> 注：此功能为实验性，需在配置文件中添加 `experimental.cron: true`，或设置环境变量 `QWEN_CODE_ENABLE_CRON=1` 后使用。

## 📊 优化改进

- **/review 增强**：审查结果支持验证和误报控制，还能直接生成 PR 评论格式（[#2687](https://github.com/QwenLM/qwen-code/pull/2687)）
- **/btw 侧问增强**：支持 Ctrl+C/D 取消、提示词改进，侧问打断更方便（[#2776](https://github.com/QwenLM/qwen-code/pull/2776)）
- **Follow-up 建议默认禁用**：不再每次回答完都弹建议，按需手动开启（[#2954](https://github.com/QwenLM/qwen-code/pull/2954)）
- **权限判断更准确**：非核心工具能正确绕过白名单，权限弹窗不再误拦截（[#2843](https://github.com/QwenLM/qwen-code/pull/2843)）
- **VS Code 新建标签页继承模型选择**：新建聊天标签页时自动沿用之前选的模型（[#2802](https://github.com/QwenLM/qwen-code/pull/2802)）
- **SSH 高延迟下终端响应不丢失**：远程连接时 AI 回答不再漏字或截断（[#2718](https://github.com/QwenLM/qwen-code/pull/2718)）
- **文件编辑保留原始换行符**：CRLF/LF 格式不再被强制转换，Git diff 不乱（[#2707](https://github.com/QwenLM/qwen-code/pull/2707)）
- **输入拖拽检测修复**：解决鼠标拖拽选中文字时的输入延迟问题（[#2837](https://github.com/QwenLM/qwen-code/pull/2837)）
- **output-language.md 启动时不再被覆盖**：自定义输出语言配置不会丢失（[#2842](https://github.com/QwenLM/qwen-code/pull/2842)）
- **Ctrl+Y 跳过限流重试**：遇到 API 限流时不再干等，按 Ctrl+Y 直接跳过（[#2420](https://github.com/QwenLM/qwen-code/pull/2420)）
- **MCP 自动重连**：连接断开后自动重连，不用手动重启（[#2428](https://github.com/QwenLM/qwen-code/pull/2428)）
- **Hooks 移除 experimental 标志**：Hooks 系统正式可用，不再有"实验性"提示（[#2781](https://github.com/QwenLM/qwen-code/pull/2781)）
- **扩展安装支持 NPM 注册表**：装扩展不仅支持 GitHub，还能直接从 npm 安装，跟装 npm 包一样简单（[#2719](https://github.com/QwenLM/qwen-code/pull/2719)）
