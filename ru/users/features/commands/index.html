Команды

В этом документе подробно описаны все команды, поддерживаемые Qwen Code, которые помогут вам эффективно управлять сеансами, настраивать интерфейс и контролировать его поведение.

Команды Qwen Code активируются через определённые префиксы и делятся на три категории:

Тип префиксаОписание функцииТипичный сценарий использования
Команды с косой чертой (/)Управление Qwen Code на метауровнеУправление сеансами, изменение настроек, помощь
Команды с «собакой» (@)Быстрая вставка локального содержимого файлов в диалогРазрешение ИИ анализировать указанные файлы или код в каталогах
Команды с восклицательным знаком (!)Прямое взаимодействие с системной оболочкойВыполнение системных команд, таких как git status, ls и т.д.

1. Команды с косой чертой (/)

Команды с косой чертой используются для управления сеансами Qwen Code, интерфейсом и базовым поведением.

1.1 Управление сеансом и проектом

Эти команды помогают сохранять, восстанавливать и обобщать результаты работы.

КомандаОписаниеПримеры использования
/initАнализирует текущий каталог и создаёт начальный контекстный файл/init
/summaryГенерирует сводку проекта на основе истории диалога/summary
/compressЗаменяет историю диалога сводкой для экономии токенов/compress или /summarize
/compress-fastБыстрое сжатие без ИИ — удаляет старые выводы инструментов и «размышления»/compress-fast
/resumeВозобновляет предыдущий сеанс диалога/resume или /continue
/recapГенерирует однострочную сводку текущего сеанса прямо сейчас/recap
/restoreОткатывает файлы проекта к контрольной точке до выполнения вызова инструмента/restore (список) или /restore <ID>
/deleteУдаляет предыдущий сеанс/delete
/branchСоздаёт ответвление текущего диалога в новый сеанс/branch
/forkЗапускает фонового агента, наследуя полный диалог/fork <директива>
/rewindОтматывает диалог к предыдущему шагу/rewind или /rollback
/exportЭкспортирует историю сеанса в файл/export html, /export md, /export json, /export jsonl
/renameПереименовывает или добавляет тег текущему сеансу/rename My Feature или /tag
Note

/summarize — это псевдоним для /compress (он сжимает историю диалога — деструктивная операция). Чтобы вместо этого создать неразрушающую сводку проекта, используйте /summary.

1.2 Управление интерфейсом и рабочей областью

Команды для настройки внешнего вида интерфейса и рабочего окружения.

КомандаОписаниеПримеры использования
/clearОчищает историю диалога и освобождает контекст/clear, /reset, /new
/contextПоказывает разбивку использования контекстного окна/context
→ detailПоказывает разбивку использования контекста по элементам/context detail
/historyУправляет настройками отображения истории и её видимостью/history collapse-on-resume, /history expand-on-resume, /history expand-now
/diffОткрывает интерактивное окно просмотра различий, показывающее неотслеживаемые изменения и изменения по шагам. Используйте ←/→ для переключения между текущим git diff и отдельными шагами диалога, ↑/↓ для просмотра файлов/diff
/themeИзменяет визуальную тему Qwen Code/theme
/vimВключает/выключает режим редактирования Vim в поле ввода/vim
/voiceВключает/выключает голосовой ввод/voice, /voice hold, /voice tap, /voice off, /voice status
/directoryУправляет многокаталоговой рабочей областью/dir add ./src,./tests, /dir show
/cdПеремещает текущий сеанс в новый рабочий каталог/cd ../other-project
/editorОткрывает диалог выбора поддерживаемого редактора/editor
/statuslineОткрывает интерактивный диалог выбора предустановок статусной строки/statusline
/statusline <text>Генерирует статусную строку в командном режиме через агента/statusline show model and git branch
/terminal-setupНастраивает привязки клавиш терминала для многострочного ввода/terminal-setup

1.3 Языковые настройки

Команды для управления языком интерфейса и вывода.

КомандаОписаниеПримеры использования
/languageПросмотр или изменение языковых настроек/language
→ ui [язык]Установить язык интерфейса/language ui zh-CN
→ output [язык]Установить язык вывода LLM/language output Chinese
  • Доступные встроенные языки интерфейса: zh-CN (Упрощённый китайский), en-US (Английский), ru-RU (Русский), de-DE (Немецкий), ja-JP (Японский), pt-BR (Португальский – Бразилия), fr-FR (Французский), ca-ES (Каталанский).
  • Примеры языка вывода: Chinese, English, Japanese и т.д.

1.4 Управление инструментами и моделями

Команды для управления инструментами и моделями ИИ.

КомандаОписаниеПримеры использования
/mcpВыводит список настроенных MCP-серверов и инструментов/mcp, /mcp desc, /mcp nodesc, /mcp schema
/import-configИмпортирует MCP-серверы из конфигураций Claude/import-config all, /import-config claude-code, /import-config claude-desktop --scope user|project
/toolsПоказывает текущий список доступных инструментов/tools, /tools desc
/skillsВыводит список и запускает доступные навыки/skills, /skills <имя>
/planПереключается в режим планирования или выходит из него/plan, /plan <задача>, /plan exit
/approval-modeИзменяет режим одобрения инструментов (только для текущего сеанса)/approval-mode, /approval-mode auto-edit
→ planТолько анализ, без выполнения (безопасная проверка)/approval-mode plan
→ defaultТребуется одобрение для изменений (повседневное использование)/approval-mode default

1.6 Побочный вопрос (/btw)

Команда /btw позволяет задавать быстрые побочные вопросы, не прерывая и не влияя на основной ход диалога.

КомандаОписание
/btw <ваш вопрос>Задать быстрый побочный вопрос
?btw <ваш вопрос>Альтернативный синтаксис для побочных вопросов

Как это работает:

  • Побочный вопрос отправляется как отдельный вызов API с контекстом недавнего диалога (до последних 20 сообщений).
  • Ответ отображается над полем ввода — вы можете продолжать печатать, пока ждёте.
  • Основной диалог не блокируется — он продолжается независимо.
  • Ответ на побочный вопрос не становится частью истории основного диалога.
  • Ответы отображаются с полной поддержкой Markdown (блоки кода, списки, таблицы и т.д.).

Горячие клавиши (интерактивный режим):

Сочетание клавишДействие
EscapeОтменить (во время загрузки) или закрыть (после завершения)
Space или EnterЗакрыть ответ (когда поле ввода пусто)
Ctrl+C или Ctrl+DОтменить выполняющийся побочный вопрос

Пример:

(Пока основной диалог посвящён рефакторингу кода)

> /btw Какая разница между let и var в JavaScript?

  ╭──────────────────────────────────────────╮
  │ /btw Какая разница между let и var в     │
  │     JavaScript?                          │
  │                                          │
  │ + Отвечаю...                             │
  │ Нажмите Escape, Ctrl+C или Ctrl+D,       │
  │ чтобы отменить                           │
  ╰──────────────────────────────────────────╯
  > (Поле ввода остаётся активным — продолжайте печатать)

(После получения ответа)

  ╭──────────────────────────────────────────╮
  │ /btw Какая разница между let и var в     │
  │     JavaScript?                          │
  │                                          │
  │ `let` имеет блочную область видимости,   │
  │ а `var` — функциональную. `let` появился │
  │ в ES6 и не «всплывает» так же, как `var`.│
  │                                          │
  │ Нажмите Space, Enter или Escape,         │
  │ чтобы закрыть                            │
  ╰──────────────────────────────────────────╯
  > (Поле ввода всё ещё активно)

Поддерживаемые режимы выполнения:

Tip

Используйте /btw, когда нужен быстрый ответ, не отвлекаясь от основной задачи. Это особенно полезно для уточнения концепций, проверки фактов или получения быстрых пояснений, оставаясь сфокусированным на основном рабочем процессе.

1.7 Итоги сессии (/recap)

Команда /recap генерирует краткий обзор текущей сессии в стиле «на чём вы остановились», чтобы вы могли возобновить старый разговор без прокрутки страниц истории.

КомандаОписание
/recapСгенерировать и показать однострочные итоги сессии

Как это работает:

  • Использует настроенную быструю модель (параметр fastModel), если она доступна, иначе — основную модель сессии. Небольшая дешёвая модель вполне подходит для итогов.
  • Последние сообщения разговора (до 30 сообщений, только текст — вызовы инструментов и ответы инструментов отфильтровываются) отправляются модели с кратким системным промптом.
  • Итоги отображаются тусклым цветом с префиксом ❯, чтобы они выделялись на фоне реальных ответов ассистента.
  • Команда отказывается с сообщением об ошибке, если выполняется ответ модели или обрабатывается другая команда. Если нет полезного разговора или не удалось сгенерировать итоги, /recap показывает краткое информационное сообщение вместо итогов — ручная команда всегда что-то возвращает.

Автоматический запуск при возвращении после отсутствия:

Если терминал был свёрнут 5+ минут и снова получает фокус, автоматически генерируются и показываются итоги (только когда не выполняется ответ модели; в противном случае ожидается завершение текущего ответа, а затем итоги запускаются). В отличие от ручной команды, автоматический запуск полностью молчалив при ошибке: если генерация не удалась или нечего обобщать, сообщение не добавляется в историю. Управляется параметром general.showSessionRecap (по умолчанию: false); ручная команда /recap всегда работает независимо от этого параметра.

Пример:

> /recap

❯ Рефакторинг loopDetectionService.ts для устранения OOM при длительных сессиях,
  вызванного неограниченным streamContentHistory и contentStats. Следующий шаг —
  реализовать вариант B (LRU скользящее окно с FNV-1a) после подтверждения.
Tip

Настройте быструю модель через /model --fast <модель> (например, qwen3-coder-flash), чтобы сделать /recap быстрым и дешёвым. Установите general.showSessionRecap в true, чтобы включить автоматический запуск; ручная команда /recap всегда работает независимо от этого параметра.

1.8 Просмотр изменений (/diff)

Команда /diff открывает интерактивный просмотр изменений, показывая незакоммиченные изменения и изменения по шагам. Используйте ←/→ для переключения между текущим git diff и отдельными шагами разговора, ↑/↓ для навигации по файлам и Enter для просмотра встроенного diff.

Как это работает:

В интерактивном режиме /diff открывает диалог с выбором источника в верхней части:

  • Current — рабочее дерево vs HEAD (git diff HEAD). Показывает все незакоммиченные изменения, включая staged, unstaged и неотслеживаемые файлы.
  • T1, T2, T3, … — изменения по шагам, по одной вкладке на каждый шаг модели, который изменял файлы. Самые последние шаги отображаются первыми. Каждая вкладка показывает превью оригинального промпта для контекста.

В списке файлов отображается статистика по каждому файлу (добавлено/удалено строк) с тегами для особых состояний (new, deleted, untracked, binary, truncated, oversized). Нажмите Enter на файле, чтобы просмотреть его встроенный diff с подсветкой синтаксиса.

Изменения по шагам требуют, чтобы сохранение контрольных точек файлов было включено (по умолчанию включено в интерактивном режиме). Если сохранение контрольных точек выключено, доступен только источник «Current».

Горячие клавиши:

КлавишаДействие
← / →Переключение между источниками (Current / T1 / T2…)
↑ / ↓Навигация по списку файлов
j / kНавигация по списку файлов (в стиле vim)
EnterПросмотр встроенного diff для выбранного файла
← / EscВозврат к списку файлов из просмотра diff
EscЗакрыть диалог

Пример:

┌ /diff · Шаг 3 "рефакторинг auth middleware" ──── 3 файла +45 -12 ┐
│                                                                     │
│ ◀ Current · T3 · T2 · T1 ▶                                         │
│                                                                     │
│ › src/utils/parser.ts                              +30 -8           │
│   src/utils/parser.test.ts                         +12 -2           │
│   README.md                                        +3 -2            │
│                                                                     │
│ ←/→ источник · ↑/↓ файл · Enter просмотр · Esc закрыть             │
└─────────────────────────────────────────────────────────────────────┘

Неинтерактивный режим:

В безголовом (--prompt) или неинтерактивном контексте /diff выводит текстовую сводку рабочего дерева vs HEAD. Навигация по шагам недоступна.

3 файла изменено, +45 / -12
  +30  -8  src/utils/parser.ts
  +12  -2  src/utils/parser.test.ts
   +3  -2  README.md

5. Подкоманды CLI

Эти команды запускаются из оболочки как qwen <подкоманда> перед началом интерактивного сеанса.

Управление сеансами

КомандаОписаниеПримеры использования
qwen sessions listСписок недавних сеансов беседыqwen sessions list, qwen sessions list --json --limit 50

qwen sessions list

Выводит список ваших недавних сеансов Qwen Code с метаданными.

Флаги:

ФлагТипПо умолчаниюОписание
--jsonbooleanfalseВывод в формате JSON Lines (один объект JSON на строку)
--limitnumber20Максимальное количество отображаемых сеансов

Вывод, удобочитаемый для человека (по умолчанию):

Таблица со столбцами: SESSION ID, STARTED (метка времени UTC), TITLE, BRANCH, PROMPT.

Вывод в JSON (--json):

Выводит JSON Lines в stdout. Каждая строка представляет собой объект JSON с полями:

sessionId, startTime, mtime, prompt, gitBranch, customTitle, titleSource, filePath, cwd

Подсказка «есть ещё сеансы» выводится через stderr, чтобы передача в jq оставалась безопасной.

Примеры:

# Show last 20 sessions (default)
qwen sessions list
 
# Show last 50 sessions
qwen sessions list --limit 50
 
# Output as JSON for scripting
qwen sessions list --json | jq .
