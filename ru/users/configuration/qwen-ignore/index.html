Игнорирование файлов

Этот документ предоставляет обзор функции Qwen Ignore (.qwenignore) в Qwen Code. Qwen Code также распознает пользовательские файлы игнорирования, настроенные с помощью context.fileFiltering.customIgnoreFiles, которые по умолчанию устанавливаются на файлы совместимости .agentignore и .aiignore.

Qwen Code включает возможность автоматически игнорировать файлы, подобно .gitignore (используемому Git). Добавление путей в .qwenignore или настроенный пользовательский файл игнорирования исключит их из инструментов, поддерживающих эту функцию, хотя они останутся видимыми для других сервисов (например, Git).

Как это работает

Когда вы добавляете путь в один из этих файлов игнорирования, инструменты, которые учитывают правила Qwen Ignore, будут исключать соответствующие файлы и директории из своих операций. Например, при использовании команды read_many_files любые пути из .qwenignore или настроенных пользовательских файлов игнорирования будут автоматически исключены.

В основном эти файлы игнорирования следуют соглашениям файлов .gitignore:

  • Пустые строки и строки, начинающиеся с #, игнорируются.
  • Поддерживаются стандартные glob-шаблоны (такие как *, ? и []).
  • Символ / в конце соответствует только директориям.
  • Символ / в начале привязывает путь относительно файла игнорирования.
  • ! отрицает шаблон.

Вы можете обновлять эти файлы игнорирования в любое время. Чтобы применить изменения, необходимо перезапустить сессию Qwen Code.

Как использовать файлы игнорирования

ШагОписание
Включить правила игнорированияСоздайте .qwenignore, файл по умолчанию (.agentignore / .aiignore) или настроенный пользовательский файл игнорирования в корневой директории вашего проекта
Добавить правила игнорированияОткройте файл игнорирования и добавьте пути для игнорирования, например: /archive/ или apikeys.txt

По умолчанию Qwen Code читает .qwenignore, .agentignore и .aiignore. Чтобы использовать другой пользовательский файл игнорирования, настройте:

{
  "context": {
    "fileFiltering": {
      "customIgnoreFiles": [".cursorignore"]
    }
  }
}

.qwenignore всегда включается, когда включена опция context.fileFiltering.respectQwenIgnore. Пути к пользовательским файлам игнорирования указываются относительно корня проекта.

Примеры файлов игнорирования

Вы можете использовать любой поддерживаемый файл игнорирования для игнорирования директорий и файлов:

# Исключить вашу директорию /packages/ и все поддиректории
/packages/

# Исключить ваш файл apikeys.txt
apikeys.txt

Вы можете использовать подстановочные знаки в вашем файле игнорирования с помощью *:

# Исключить все .md файлы
*.md

Наконец, вы можете исключать файлы и директории из исключения с помощью !:

# Исключить все .md файлы, кроме README.md
*.md
!README.md

Чтобы удалить пути из файла игнорирования, удалите соответствующие строки.
